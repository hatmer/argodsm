enum argo::allocation
strong

Enumeration class with parameters for the C++ allocation functions.

Enumerator
initialize 

Initialize the element(s) allocated.

If this parameter is given to the allocation function, then the function will initialize the element(s) allocated. This is done by calling a placement new expression on the memory allocated. This option is enabled by default for non-trivial types.

no_initialize 

Do not initialize the element(s) allocated.

The opposite of the initialize parameter. It is enabled by default for trivial types.

deinitialize 

Destruct (deinitialize) the element(s) deallocated.

This parameters is the equivalent of initialize for the delete family of functions.

no_deinitialize 

Do not destruct (deinitialize) the deallocated element(s)

The opposite of deinitialize.

synchronize 

Make the allocation a synchronization point.

This will cause the allocator to synchronize with the other nodes. This is the default behaviour in the collective allocator if initialize is also enabled.

no_synchronize 

Do not make the allocation a synchronization point.

The opposite of synchronize. Enabled by default in the collective allocator if no_initialize is enabled, and always for the dynamic allocator.
